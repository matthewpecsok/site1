🎯 Learning Objectives

By the end of today's session, you will be able to:

  • Understand the structure and purpose of HTML
  • Create semantic HTML documents
  • Apply CSS for styling and layout
  • Use responsive design principles
  • Implement modern web development practices

📚 What We'll Cover Today

HTML Essentials

  • Document structure & DOCTYPE
  • Semantic HTML5 elements
  • Forms and input validation
  • Accessibility best practices
  • Meta tags and SEO basics

CSS Fundamentals

  • Selectors and specificity
  • Box model and positioning
  • Flexbox and Grid layouts
  • Responsive design with media queries
  • CSS animations and transitions

💻 Code Example: Basic HTML Structure

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>My First Page</title> <style> body { font-family: Arial, sans-serif; margin: 0; padding: 20px; } .container { max-width: 800px; margin: 0 auto; } h1 { color: #CC0000; text-align: center; } </style> </head> <body> <div class="container"> <header> <h1>Welcome to Web Development</h1> </header> <main> <p>This is a simple HTML page with embedded CSS.</p> </main> </div> </body> </html>

🎨 CSS Grid Example

.grid-container { display: grid; grid-template-columns: 1fr 2fr 1fr; grid-gap: 20px; padding: 20px; } .grid-item { background: #f0f0f0; padding: 20px; border-radius: 8px; text-align: center; } /* Responsive design */ @media (max-width: 768px) { .grid-container { grid-template-columns: 1fr; } }

🏃‍♂️ Hands-On Activities

Lab Exercise 1: Personal Portfolio Page

Create a personal portfolio webpage that includes:

  • Semantic HTML structure (header, nav, main, footer)
  • CSS Grid or Flexbox layout
  • Responsive design for mobile devices
  • At least 3 different font weights and sizes
  • A color scheme inspired by University of Utah branding

🌟 Pro Tips

  • Always validate your HTML at validator.w3.org
  • Use semantic elements for better accessibility and SEO
  • Mobile-first approach - design for small screens first
  • CSS custom properties (variables) make maintaining styles easier
  • Practice, practice, practice! The best way to learn is by building
